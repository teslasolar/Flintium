Perspective Demo renders the same view JSONs shipped in the project zip

The Flintium project zip embeds Perspective views as JSON. This page reads the same JSONs from pages/data/perspective/views/ and renders them in the browser using a tiny Perspective-compatible runtime that supports ia.container.coord, ia.display.label, ia.display.table, and ia.display.markdown. runScript() bindings are routed through a browser-side mock of flintium.tools, so the tables populate from the live UDT and instance indexes.

How this works

  1. tools/build_perspective_export.py generates the same view JSONs twice — once into the Ignition project zip under com.inductiveautomation.perspective/views/..., and once into pages/data/perspective/views/... for this page.
  2. The renderer (pages/assets/perspective/renderer.js) walks the view tree, dispatches each node to a component module, and resolves propConfig bindings.
  3. Expression bindings of the form runScript("flintium.tools.list_udts", 0) are intercepted by tools-mock.js, which serves the same data the gateway-side Jython implementation would return.
  4. The label-on-the-Home view fires system.perspective.navigate('Flintium/UDTBrowser'); we map that to a custom perspective:navigate DOM event that swaps the current view.
